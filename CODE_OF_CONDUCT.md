# Code of Conduct

## Our Pledge

We want HOOTNER to be a welcoming, professional, and safe community for everyone who participates, regardless of background, identity, or experience.

## Our Standards

Examples of behavior that helps create a positive environment include:

- Being respectful and constructive
- Assuming good intent and giving thoughtful feedback
- Focusing on what is best for users and the project
- Being mindful of security, privacy, and safety

Examples of unacceptable behavior include:

- Harassment, bullying, or intimidation
- Discriminatory or demeaning comments
- Publishing other people’s private information without permission
- Threats or incitement of violence
- Knowingly introducing malicious code or security backdoors

## Security and Responsible Disclosure

If you believe you found a security vulnerability, do not open a public issue. Please follow the reporting instructions in SECURITY.md.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.

If you experience or witness unacceptable behavior, report it to:

- support@hootner.com

Include relevant context (links, screenshots, logs) where possible. Reports will be handled as confidentially as practical.

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues and pull requests
- Discussions, reviews, and code comments
- Community channels associated with the project

## Attribution

This policy is inspired by common open source community standards. It is intentionally concise to fit HOOTNER’s security-focused development workflow.
