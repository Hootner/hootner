🔐 HOOTNER User Authentication Guide

📧 How Users Log In (EMAIL-BASED)

✅ Primary Login Method: Email + Password

Users log in with their EMAIL ADDRESS, not mailing address!

  • Username: Email address (e.g., user@example.com)
  • Password: Must meet security requirements
  • Verification: Email verification required

🚀 User Registration Process

Step 1: User Signs Up

Sign Up Form:
Email: john.doe@example.com
Password: SecurePass123!
Confirm Password: SecurePass123!

Step 2: Email Verification

AWS Cognito sends verification email to user's inbox:

Email Subject: "Verify your HOOTNER account"
Content: Click link or enter verification code: 123456

Step 3: Account Activated

✅ User can now log in and trigger render jobs!

🔑 Password Requirements

Security Policy (from template.yaml):
  • 🔢 Minimum 8 characters
  • 🔠 At least 1 uppercase letter
  • 🔡 At least 1 lowercase letter
  • 🔢 At least 1 number
  • 🔣 At least 1 symbol (!@#$%^&*)

💻 Login Process (Frontend)

Option 1: Custom Login Form

// React/JavaScript Login
import { Auth } from 'aws-amplify';

const login = async (email, password) => {
    try {
        const user = await Auth.signIn(email, password);
        console.log('Logged in:', user);
        
        // Get access token for API calls
        const session = await Auth.currentSession();
        const token = session.getAccessToken().getJwtToken();
        
        // Now user can trigger render jobs!
        
    } catch (error) {
        console.error('Login failed:', error);
    }
};

Option 2: Cognito Hosted UI

AWS provides a ready-made login page:
URL: https://your-domain.auth.us-east-1.amazoncognito.com/login

Features:
  • Professional login/signup forms
  • Password reset functionality
  • Social login (Google, Facebook) - optional
  • Multi-factor authentication - optional

🔄 Complete Authentication Flow

1. User visits HOOTNER app → Sees "Login Required" message
2. User clicks "Login" → Redirected to login form
3. User enters email + password → Cognito validates credentials
4. Login successful → User gets JWT access token
5. User triggers render job → Token sent to Lambda function
6. Lambda verifies token → If valid, starts GPU render job

🛡️ Security Features

What's Protected:

  • 🚫 No anonymous render job submissions
  • 🔒 Each user only sees their own jobs
  • 💰 Prevents unauthorized GPU usage ($1-3/hour)
  • 📊 Usage tracking per user account
  • ⏰ Token expiration (automatic logout)

📱 User Experience

For Users, Login is Simple:

  1. First time: Create account with email
  2. Verify email: Click link in inbox
  3. Regular use: Just email + password
  4. Render jobs: Click "Start Render" (authenticated automatically)
⚠️ Important: Users need EMAIL ACCESS to verify their account initially. After that, just email + password for login.
